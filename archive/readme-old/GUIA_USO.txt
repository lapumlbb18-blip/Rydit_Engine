================================================================================
                              CONFIGURACIÓN INICIAL
================================================================================

1. INSTALAR SCCACHE (¡RECOMENDADO! - 12-17x más rápido)
   ----------------------------------------------------------------------------
   bash scripts/setup-sccache.sh
   source ~/.bashrc
   ----------------------------------------------------------------------------
   Uso: Configurar caché de compilación para builds rápidos
   Beneficio real medido:
     - Primer build:  ~60-90 segundos
     - Segundo build: ~3-5 segundos (¡17x más rápido!)

2. VERIFICAR INSTALACIÓN DE RAYLIB
   ----------------------------------------------------------------------------
   pkg-config --libs --cflags raylib
   ----------------------------------------------------------------------------
   Salida esperada:
     -I/data/data/com.termux/files/usr/include 
     -L/data/data/com.termux/files/usr/lib -lraylib

================================================================================
                              COMANDOS BÁSICOS
================================================================================

1. VERIFICAR ERRORES DE COMPILACIÓN (RÁPIDO)
   ----------------------------------------------------------------------------
   cargo check
   ----------------------------------------------------------------------------
   Uso: Verificar sintaxis y errores sin compilar completamente
   Tiempo: ~1-5 segundos
   RAM: ~200-300 MB

2. COMPILAR PROYECTO
   ----------------------------------------------------------------------------
   cargo build
   ----------------------------------------------------------------------------
   Uso: Compilar todo el proyecto en modo debug
   Output: target/debug/rydit-rs
   Tiempo (1ra vez): ~60-90 segundos
   Tiempo (2da vez): ~3-5 segundos (con sccache)
   RAM pico: ~2.0 GB

3. COMPILAR EN MODO RELEASE (OPTIMIZADO)
   ----------------------------------------------------------------------------
   cargo build --release
   ----------------------------------------------------------------------------
   Uso: Compilar con optimizaciones máximas
   Output: target/release/rydit-rs
   Nota: Usa opt-level = "z" (tamaño), LTO activado
   Tiempo: 2-5 minutos
   RAM pico: ~2.5 GB

4. EJECUTAR EL PROGRAMA
   ----------------------------------------------------------------------------
   cargo run --bin rydit-rs
   ----------------------------------------------------------------------------
   Uso: Compilar y ejecutar inmediatamente
   Tiempo: 3-60 segundos (depende de caché)

5. EJECUTAR BINARIO DIRECTAMENTE
   ----------------------------------------------------------------------------
   ./target/debug/rydit-rs
   ----------------------------------------------------------------------------
   Uso: Ejecutar binario ya compilado

================================================================================
                              COMANDOS DE MANTENIMIENTO
================================================================================

6. LIMPIAR PROYECTO
   ----------------------------------------------------------------------------
   cargo clean
   ----------------------------------------------------------------------------
   Uso: Eliminar carpeta target/ y artifacts de compilación
   Nota: Libera ~140-200 MB de espacio

7. ACTUALIZAR DEPENDENCIAS
   ----------------------------------------------------------------------------
   cargo update
   ----------------------------------------------------------------------------
   Uso: Actualizar Cargo.lock con últimas versiones compatibles

8. VER DEPENDENCIAS
   ----------------------------------------------------------------------------
   cargo tree
   ----------------------------------------------------------------------------
   Uso: Ver árbol de dependencias del proyecto

9. VERIFICAR FORMATEO
   ----------------------------------------------------------------------------
   cargo fmt --check
   ----------------------------------------------------------------------------
   Uso: Verificar si el código está formateado correctamente

10. FORMATEAR CÓDIGO
    ----------------------------------------------------------------------------
    cargo fmt
    ----------------------------------------------------------------------------
    Uso: Formatear automáticamente todo el código Rust

11. VER USO DE SCCACHE
    ----------------------------------------------------------------------------
    sccache --show-stats
    ----------------------------------------------------------------------------
    Uso: Ver estadísticas de caché de compilación

12. LIMPIAR CACHE DE SCCACHE
    ----------------------------------------------------------------------------
    sccache --zero-stats
    ----------------------------------------------------------------------------
    Uso: Reiniciar estadísticas de caché

================================================================================
                              SOLUCIÓN DE PROBLEMAS
================================================================================

PROBLEMA: Error de compilación con raylib
SOLUCIÓN: Verificar que raylib esté instalado en Termux
          $ pkg install raylib
          $ pkg-config --libs --cflags raylib

PROBLEMA: Error "edition 2024 not found"
SOLUCIÓN: Usar edition = "2021" en Cargo.toml

PROBLEMA: Warning de resolver
SOLUCIÓN: Agregar resolver = "3" en el workspace Cargo.toml

PROBLEMA: Variables no usadas
SOLUCIÓN: Prefijar con underscore: _variable

PROBLEMA: Build lento
SOLUCIÓN: Instalar sccache: bash scripts/setup-sccache.sh

PROBLEMA: Poco espacio en disco
SOLUCIÓN: $ cargo clean && sccache --zero-stats

PROBLEMA: Error de memoria insuficiente
SOLUCIÓN: El proyecto ya usa codegen-units = 1 para reducir RAM
          Cerrar otras aplicaciones durante el build

PROBLEMA: sccache no funciona
SOLUCIÓN: Verificar variable de entorno:
          $ echo $RUSTC_WRAPPER
          Debe mostrar: sccache
          Si no: source ~/.bashrc
