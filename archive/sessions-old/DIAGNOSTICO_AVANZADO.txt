                    3. PRONÓSTICO DE ERRORES A FUTURO
================================================================================

ERROR SEGURO #1: Memory Leak en Executor
-----------------------------------------
PROBABILIDAD: 80%
CUÁNDO: Cuando agregues funciones con scopes
POR QUÉ: HashMap nunca se limpia, variables se acumulan
SOLUCIÓN: Implementar stack de scopes con cleanup
PREVENCIÓN: Agregar tests de estrés de memoria

ERROR SEGURO #2: Stack Overflow en Parser
------------------------------------------
PROBABILIDAD: 60%
CUÁNDO: Expresiones muy anidadas o recursivas
POR QUÉ: Parser recursivo sin límite de profundidad
SOLUCIÓN: Parser iterativo o límite de profundidad
PREVENCIÓN: Test con expresión de 1000+ tokens

ERROR SEGURO #3: Race Condition en REPL
----------------------------------------
PROBABILIDAD: 40%
CUÁNDO: Si agregas concurrencia/threads
POR QUÉ: Executor no es thread-safe
SOLUCIÓN: Mutex o Arc alrededor del estado
PREVENCIÓN: Mantener REPL single-threaded

ERROR SEGURO #4: Error de Tipos en Runtime
------------------------------------------
PROBABILIDAD: 90%
CUÁNDO: Cuando sumes string + número
POR QUÉ: No hay chequeo de tipos en compile-time
SOLUCIÓN: Sistema de tipos básico o error claro
PREVENCIÓN: Tests de type errors

ERROR SEGURO #5: Parser de Expresiones Roto
-------------------------------------------
PROBABILIDAD: 70%
CUÁNDO: Agregues precedencia de operadores compleja
POR QUÉ: 5-level parser es frágil a cambios
SOLUCIÓN: Parser generator o Pratt parsing
PREVENCIÓN: Tests exhaustivos de precedencia

ERROR SEGURO #6: raylib no funciona en Android
----------------------------------------------
PROBABILIDAD: 50%
CUÁNDO: Intentes mostrar ventana gráfica
POR QUÉ: raylib en Termux es limitado (sin contexto gráfico real)
SOLUCIÓN: Usar TUI (ratatui) en vez de gráficos
PREVENCIÓN: Prototipar en desktop primero

ERROR SEGURO #7: Build Times Insoportables
------------------------------------------
PROBABILIDAD: 95%
CUÁNDO: Proyecto crezca a 5000+ líneas
POR QUÉ: Rust compila lento, sccache ayuda pero no magic
SOLUCIÓN: Más crates, incremental compilation
PREVENCIÓN: Mantener crates pequeños
