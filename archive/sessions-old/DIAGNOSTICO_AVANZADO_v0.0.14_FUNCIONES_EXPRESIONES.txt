CAMBIO #1: &Executor → &mut Executor
-------------------------------------
ARCHIVO: crates/rydit-rs/src/main.rs

ANTES:
```rust
fn evaluar_expr(expr: &Expr, executor: &Executor, ...) -> Valor
```

DESPUÉS:
```rust
fn evaluar_expr(expr: &Expr, executor: &mut Executor, ...) -> Valor
```

IMPACTO:
- ✅ Permite ejecutar funciones dentro de expresiones
- ✅ Scopes se crean/eliminan correctamente
- ⚠️ Requiere actualizar 50+ llamadas
- ⚠️ Borrow checker requirió clonado de datos

CAMBIO #2: Implementación de Funciones de Usuario
-------------------------------------------------
CÓDIGO AGREGADO (~50 líneas):
```rust
// Función de usuario - ejecutar y capturar retorno
let func_data = funcs.get(name).map(|(p, b)| (p.clone(), b.clone()));

if let Some((params, body)) = func_data {
    // Evaluar argumentos
    let mut arg_values = vec![];
    for arg in args {
        arg_values.push(evaluar_expr(arg, executor, funcs));
    }

    // Crear scope local
    executor.push_scope();
    
    // Mapear parámetros → argumentos
    for (i, param) in params.iter().enumerate() {
        if i < arg_values.len() {
            executor.guardar_local(param, arg_values[i].clone());
        }
    }

    // Ejecutar body y capturar retorno
    // ...
    
    executor.pop_scope();
    return return_value.unwrap_or(Valor::Vacio);
}
```

DEBILIDADES RESTANTES:
----------------------

1. **FUNCIÓN EN MODO GRÁFICO SIN SCOPE** ⭐⭐⭐
   - ejecutar_stmt_gfx no implementa scopes para funciones
   - Funciones en gráficos no tienen scope local real
   
   IMPACTO: Medio - afecta juegos complejos
   
   SOLUCIÓN: Implementar mismo patrón que evaluar_expr

2. **CLONADO DE DATOS DE FUNCIÓN** ⭐⭐⭐
   - Clonar params y body tiene costo de performance
   - Podría optimizarse con referencias inteligentes
   
   IMPACTO: Bajo - negligible para scripts pequeños
   
   SOLUCIÓN FUTURA: Rc<RefCell<>> para shared state

3. **SIN SISTEMA DE TIPOS** ⭐⭐
   - `sumar("texto", 5)` → Error en runtime, no compile time
   
   IMPACTO: Medio - errores se detectan tarde
   
   SOLUCIÓN: Sistema de tipos opcional (v0.2.0)

================================================================================
                         📋 LECCIONES APRENDIDAS
================================================================================

1. **&mut ES ESENCIAL PARA EJECUCIÓN**
   Evaluar expresiones necesita mutabilidad (scopes, variables).
   Intentar con &Executor solo limita funcionalidad.
   
   APLICACIÓN FUTURA: Usar &mut desde el diseño inicial.

2. **CLONAR PARA EVITAR BORROW CONFLICTS**
   Cuando iteras sobre HashMap y necesitas mutabilidad:
   ```rust
   let func_data = funcs.get(name).map(|(p, b)| (p.clone(), b.clone()));
   ```
   
   APLICACIÓN FUTURA: Patrón estándar para funciones.

3. **TESTS GUÍAN EL REFACTOR**
   60 tests dieron confianza para refactor masivo.
   Sin tests, el cambio habría sido imposible.
   
   APLICACIÓN FUTURA: Mantener cobertura, agregar más.

4. **DOCUMENTACIÓN ES INVERSIÓN**
   LIBRO_RYDIT.md (~400 líneas) parece tiempo perdido.
   En realidad: ahorra horas explicando a futuros usuarios.
   
   APLICACIÓN FUTURA: Documentar CADA feature nueva.

5. **CRECIMIENTO EXPONENCIAL ES REAL**
