PROBLEMA REPORTADO:
-------------------
En la sesión anterior, al ejecutar en la REPL:
  voz "hola"
  rytmo test(x) { voz x }

Ocurrió:
1. Loop infinito de prompts "rydit> rydit> rydit>..."
2. Lentitud extrema del dispositivo
3. Consumo excesivo de RAM
4. Primer error de esta naturaleza en el proyecto

CAUSA RAÍZ:
-----------
El parser entraba en loop infinito cuando encontraba un Identificador (Ident)
sin paréntesis () o corchetes [] después.

En parse_statement():
  Token::Ident(name) => self.parse_call(name.clone())

En parse_call():
  - Consumía el nombre (self.pos += 1)
  - Si no había (), retornaba Stmt::Call { name, args: vec![] }
  - El parser NO avanzaba correctamente
  - Loop infinito en parse_statement()

================================================================================
                         🔧 SOLUCIÓN IMPLEMENTADA
================================================================================

CAMBIO 1: Nueva función parse_call_or_ident()
---------------------------------------------
Reemplaza a parse_call() para manejar correctamente identificadores:

fn parse_call_or_ident(&mut self, name: String) -> Result<Option<Stmt>> {
    // Consumir el token Ident primero
    self.pos += 1;
    
    // Si hay (), es llamada a función
    if matches!(self.tokens[self.pos], Token::ParentIzq) {
        // Parsear llamada: nombre(args)
        return Ok(Some(Stmt::Call { name, args }));
    }
    
    // Si hay [], es indexación
    if matches!(self.tokens[self.pos], Token::CorcheteIzq) {
        return Ok(Some(Stmt::Expr(index_expr)));
    }
    
    // Solo "nombre" sin () o [] - NO es statement válido
    Ok(None)  // El parser continúa sin error
}

CAMBIO 2: parse_statement() actualizado
---------------------------------------
Token::Ident(name) => {
    if name == "input" {
        self.parse_input()
    } else {
        self.parse_call_or_ident(name.clone())  // ← Nueva función
    }
}

CAMBIO 3: parse_call() eliminada
--------------------------------
La función original parse_call() fue removida (ya no se usa).
