API DE RAYLIB 5.5.1:
--------------------
La versión 5.5.1 de raylib tiene una API inconsistente:
- Algunos métodos requieren `&RaylibThread` como primer parámetro
- Otros métodos NO lo requieren
- Esto causa errores de compilación en el ejemplo gráfico

EJEMPLO DE ERROR:
-----------------
```rust
// Algunos métodos requieren &thread:
rl.begin_drawing(&thread);  // ❌ Error en 5.5.1
rl.window_should_close(&thread);  // ❌ Error en 5.5.1

// Otros NO lo requieren:
rl.clear_background(color);  // ✅ Funciona
rl.is_key_pressed(key);  // ✅ Funciona
```

SOLUCIONES POSIBLES:
--------------------
1. Usar raylib versión diferente (4.x o 6.x)
2. Usar funciones de raylib directamente sin wrapper
3. Crear wrapper que maneje correctamente la API
