                         🐛 EL ERROR CRÍTICO
================================================================================

CUÁNDO OCURRIÓ:
---------------
Durante la sesión v0.0.9 (Snake Game), al probar la REPL con:
  voz "hola"
  rytmo test(x) { voz x }

SÍNTOMAS:
---------
1. Loop infinito de prompts: "rydit> rydit> rydit>..." (8402+ líneas)
2. Lentitud extrema del dispositivo Android
3. Consumo excesivo de RAM (~2GB+ pico)
4. Primer error de esta naturaleza en el proyecto

CAUSA RAÍZ TÉCNICA:
-------------------
El parser tenía un bug de diseño en parse_call():

```rust
// ❌ CÓDIGO BUGGY (v0.0.9)
fn parse_call(&mut self, name: String) -> Result<Option<Stmt>> {
    self.pos += 1; // consumir nombre
    
    // Si no hay (), igual retorna Stmt::Call
    if self.pos < self.tokens.len() && matches!(self.tokens[self.pos], Token::ParentIzq) {
        // parsea argumentos...
    }
    
    Ok(Some(Stmt::Call { name, args: vec![] }))  // ← BUG: args vacío
}
```

PROBLEMA: Cuando un Ident no tenía `()`, la función:
1. Consumía el token del nombre
2. No encontraba `(`
3. Retornaba `Stmt::Call { name, args: vec![] }`
4. El parser NO avanzaba correctamente en parse_statement()
5. Loop infinito: el mismo token se procesaba una y otra vez

                         📋 LISTA PRE-v0.1.0 (4 ÍTEMS CRÍTICOS)
================================================================================

ANTES DE ABORDAR v0.1.0 (Release Alpha), se identificaron 4 tareas críticas:

1. **FIX: SISTEMA DE SCOPES** ⚠️ CRÍTICO
   ----------------------------------------
   PROBLEMA: Funciones no tienen scope local real
   IMPACTO: Parámetros sobrescriben variables globales
   
   EJEMPLO ACTUAL (BUG):
   ```rydit
   dark.slot x = 10
   rytmo test(x) {  # Este x sobrescribe el global
       voz x  # Imprime lo que se pase, no el global
   }
   ```
   
   SOLUCIÓN NECESARIA:
   - Implementar stack de scopes en Executor
   - push_scope() al entrar a función
   - pop_scope() al salir
   - Mapear params → arg_values en scope local

   TIEMPO EST: 2-3 horas
   PRIORIDAD: Alta (afecta corrección del lenguaje)

2. **FIX: ARGUMENTOS DE FUNCIONES** ⚠️ CRÍTICO
   --------------------------------------------
   PROBLEMA: args evaluados pero no pasados como parámetros
   IMPACTO: Funciones no reciben argumentos reales
   
   EJEMPLO ACTUAL (BUG):
   ```rydit
   rytmo sumar(a, b) {
       voz a + b  # a y b no tienen los valores pasados
   }
   sumar(5, 3)  # Debería imprimir 8, pero falla
   ```
   
   SOLUCIÓN NECESARIA:
   - Evaluar args al llamar función
   - Guardar en scope local: { "a": 5, "b": 3 }
   - Executor usa scope local para resolver variables

   TIEMPO EST: 2-3 horas
   PRIORIDAD: Alta (afecta funcionalidad básica)

3. **FIX: WHILE EN MODO GRÁFICO** ⚠️ MEDIO
   ---------------------------------------
   PROBLEMA: while puede causar freeze si no hay límite
   IMPACTO: Límite artificial de 10 iteraciones
   
   EJEMPLO ACTUAL (WORKAROUND):
   ```rydit
   ryda condicion {  # Si condicion siempre es true, freeze
       # código...
       # Límite de 10 iteraciones para prevenir freeze
   }
   ```
   
   SOLUCIÓN NECESARIA:
   - Integrar while con game loop (1 iteración por frame)
   - O eliminar límite artificial y confiar en el usuario
   - O implementar break condicional

   TIEMPO EST: 1-2 horas
   PRIORIDAD: Media (workaround actual funciona)

4. **FEATURE: SNAKE GAME COMPLETO** 🎮 ALTA
   ----------------------------------------
   PROBLEMA: Snake actual tiene cuerpo incompleto
   IMPACTO: Juego no es completamente funcional
   
   ESTADO ACTUAL:
   - ✅ Cabeza de serpiente se mueve
   - ✅ Comida aparece (posición hardcoded)
   - ✅ Colisión con paredes
   - ❌ Cuerpo de serpiente (array de posiciones)
   - ❌ Comida aleatoria real
   - ❌ Colisión con la propia serpiente
   
   SOLUCIÓN NECESARIA:
   - Array de posiciones para cuerpo
   - RNG para comida aleatoria
   - Detectar colisión cabeza-vs-cuerpo

   TIEMPO EST: 3-4 horas
   PRIORIDAD: Alta (demo principal de v0.1.0)

TIEMPO TOTAL ESTIMADO: 8-12 horas
DEPENDENCIAS: Scopes → Argumentos → Snake completo
