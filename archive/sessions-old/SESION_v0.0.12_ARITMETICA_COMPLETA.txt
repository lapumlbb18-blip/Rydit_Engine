CAMBIO #1: Paréntesis en Parser
-------------------------------
ARCHIVO: crates/lizer/src/lib.rs - parse_primary()

AGREGADO:
```rust
// Paréntesis para agrupación: ( expr )
if matches!(self.tokens[self.pos], Token::ParentIzq) {
    self.pos += 1; // consumir (
    let expr = self.parse_expression()?;
    // Consumir )
    if self.pos < self.tokens.len() && matches!(self.tokens[self.pos], Token::ParentDer) {
        self.pos += 1;
        return Ok(expr);
    } else {
        return Err(RyDitError {
            kind: ErrorKind::SyntaxError,
            message: "Se esperaba ')' para cerrar paréntesis".to_string(),
            line: 1,
            column: self.pos,
            source: "(".to_string(),
        });
    }
}
```

IMPACTO:
- (2 + 3) * 4 = 20 ✅
- ((2 + 3) * (4 - 1)) = 15 ✅
- Precedencia correcta de operadores

CAMBIO #2: Funciones Builtin Aritméticas
----------------------------------------
ARCHIVO: crates/rydit-rs/src/main.rs - evaluar_expr() y evaluar_expr_gfx()

AGREGADAS:
1. sumar(a, b, ...) → Suma 2 o más números
2. restar(a, b) → Resta dos números
3. multiplicar(a, b, ...) → Multiplica 2 o más números
4. dividir(a, b) → Divide dos números (con manejo de división por cero)

IMPLEMENTACIÓN:
```rust
// Funciones aritméticas builtin
if name == "sumar" && args.len() >= 2 {
    let mut suma = 0.0;
    for arg in args {
        if let Valor::Num(n) = evaluar_expr(arg, executor) {
            suma += n;
        } else {
            return Valor::Error("sumar() requiere números".to_string());
        }
    }
    return Valor::Num(suma);
}

// ... similar para restar, multiplicar, dividir
```

IMPACTO:
- Funciones usables en expresiones: `dark.slot x = sumar(1, 2, 3) * 2`
- Manejo de errores para tipos inválidos
- División por cero detectada

================================================================================
                         ⚠️ LIMITACIONES ACTUALES
================================================================================

1. NO HAY POTENCIACIÓN
----------------------
```rydit
# No soportado aún
dark.slot cuadrado = 5 ^ 2  # ❌
dark.slot cubo = pow(2, 3)  # ❌
```

SOLUCIÓN FUTURA:
```rydit
# Función builtin pow(base, exponente)
dark.slot cuadrado = pow(5, 2)  # 25
```

2. NO HAY MÓDULO
----------------
```rydit
# No soportado aún
dark.slot residuo = 10 % 3  # ❌
```

SOLUCIÓN FUTURA:
```rydit
# Función builtin modulo(a, b)
dark.slot residuo = modulo(10, 3)  # 1
```

3. NO HAY RAÍZ CUADRADA
-----------------------
```rydit
# No soportado aún
dark.slot raiz = sqrt(16)  # ❌
```

SOLUCIÓN FUTURA:
```rydit
# Función builtin sqrt(n)
dark.slot raiz = sqrt(16)  # 4
```

4. FUNCIONES SOLO EN expresiones, NO COMO STATEMENTS
----------------------------------------------------
```rydit
# Esto SÍ funciona (en expresión)
dark.slot x = sumar(1, 2)

# Esto NO funciona (como statement solo)
sumar(1, 2)  # ❌ Error: Función no soportada en expresiones
```

NOTA: Las funciones builtin están diseñadas para usarse DENTRO de expresiones,
no como statements independientes.

================================================================================
                         💡 LECCIONES APRENDIDAS
================================================================================

1. **PARÉNTESIS SON ESENCIALES**
   - Sin paréntesis, no hay forma de alterar precedencia
   - Implementación simple en parse_primary()
   - Permite expresiones más claras

2. **FUNCIONES BUILTIN VS OPERADORES**
   - Operadores: +, -, *, / (sintaxis infija)
   - Funciones: sumar(a, b), restar(a, b) (sintaxis prefija)
   - Funciones permiten ≥2 argumentos (sumar(1,2,3,4,5))

3. **MANEJO DE ERRORES ES CRÍTICO**
   - División por cero debe detectarse
   - Tipos inválidos deben reportarse
   - Mensajes de error claros ayudan al usuario

4. **TESTS MANUALES COMPLEMENTAN AUTOMÁTICOS**
   - 59 tests automáticos cubren código
   - 20 tests manuales cubren casos de uso
   - Ambos son necesarios
