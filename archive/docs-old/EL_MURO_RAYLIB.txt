                         🧱 EL MURO TÉCNICO
================================================================================

PROBLEMA:
---------
La API de raylib-rs (5.5.1) es INCONSISTENTE:

```rust
// Algunos métodos SÍ requieren &thread:
handle.window_should_close(&thread)  // ✅
handle.begin_drawing(&thread)        // ✅

// Otros métodos NO requieren &thread:
handle.is_key_pressed(KEY_ESCAPE)           // ✅
d.draw_circle(400, 300, 50, RED)           // ✅
```

Esta inconsistencia ROMPE la filosofía:
> Rust = Arquitecto (controla)
> Raylib = Pincel (ejecuta)

INTENTO ACTUAL:
---------------
```rust
let (mut handle, thread) = init_window("RyDit", 800, 600);

while !handle.window_should_close(&thread) {  // SÍ necesita &thread
    let mut d = handle.begin_drawing(&thread);  // SÍ necesita &thread
    
    d.draw_circle(400, 300, 50, RED);  // NO necesita &thread
    d.draw_rectangle(100, 100, 100, 100, GREEN);
    
    if handle.is_key_pressed(KEY_ESCAPE) {  // NO necesita &thread
        break;
    }
}
```

ERROR DE COMPILACIÓN:
- `set_target_fps(&thread, 60)` ❌ (¿requiere o no &thread?)
- `is_key_pressed(KEY_ESCAPE)` ❌ (¿requiere o no &thread?)
- Signatures cambian entre versiones

OPCIONES:

1. **BINDING INTERNO** (Crear desde cero)
   ```rust
   // RyDit Graphics Layer
   pub mod rydit_gfx {
       pub fn circle(x: i32, y: i32, r: i32, color: &str);
       pub fn rect(x: i32, y: i32, w: i32, h: i32, color: &str);
       pub fn line(x1: i32, y1: i32, x2: i32, y2: i32, color: &str);
   }
   ```
   
   Ventaja: Control total, API consistente
   Desventaja: Más código que mantener

2. **SYNC LAYER** (Conector API)
   ```rust
   // RyDit-Rust Sync Connector
   pub struct RyditSync {
       rust_handle: RaylibHandle,
       rust_thread: RaylibThread,
   }
   
   impl RyditSync {
       pub fn draw_circle(&mut self, x, y, r, color);
       pub fn is_key_pressed(&self, key) -> bool;
   }
   ```
   
   Ventaja: Abstrae inconsistencias de raylib-rs
   Desventaja: Capa adicional de complejidad

3. **ADAPTADOR EXISTENTE** (Usar rust-raylib 4.x)
   ```toml
   [dependencies]
   rust-raylib = "0.45"  # Raylib 4.5, API más estable
   ```
   
   Ventaja: API más consistente
   Desventaja: Versión antigua de raylib

rydit-gfx/src/lib.rs:
```rust
//! RyDit Graphics Layer
//! Sincroniza Rust (arquitecto) con Raylib (pincel)

use raylib::prelude::*;

pub struct RyditGfx {
    handle: RaylibHandle,
    thread: RaylibThread,
}

impl RyditGfx {
    pub fn init(title: &str, w: i32, h: i32) -> Self {
        let (h, t) = raylib::init()
            .size(w, h)
            .title(title)
            .build();
        Self { handle: h, thread: t }
    }
    
    // API CONSISTENTE - Rust siempre controla
    pub fn should_close(&self) -> bool {
        self.handle.window_should_close(&self.thread)
    }
    
    pub fn begin_draw(&mut self) -> DrawHandle {
        self.handle.begin_drawing(&self.thread)
    }
    
    pub fn is_key(&self, key: i32) -> bool {
        self.handle.is_key_pressed(unsafe { std::mem::transmute(key) })
    }
    
    // ... más métodos con API consistente
}

pub struct DrawHandle {
    // Wrapper sobre RaylibDrawHandle
    // API simplificada para RyDit
}
```
