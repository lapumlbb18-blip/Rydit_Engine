                         🧱 EL MURO SUPERADO
================================================================================

PROBLEMA ORIGINAL:
------------------
raylib-rs 5.5.1 tiene API inconsistente:
- Algunos métodos requieren &thread
- Otros NO lo requieren
- Signatures cambian entre versiones

SOLUCIÓN rydit-gfx:
-------------------
1. Definimos constantes MANUALES para colores y teclas
2. Wrapper que unifica signatures
3. API consistente para RyDit
4. Rust SIEMPRE controla, Raylib SIEMPRE ejecuta

EJEMPLO DE USO:
---------------
```rust
use rydit_gfx::{RyditGfx, ColorRydit, Key};

fn main() {
    let mut gfx = RyditGfx::new("Mi Juego", 800, 600);
    gfx.set_target_fps(60);
    
    while !gfx.should_close() {
        // Rust decide input
        let escape = gfx.is_key_pressed(Key::Escape);
        
        // Raylib dibuja
        {
            let mut d = gfx.begin_draw();
            d.clear(ColorRydit::Negro);
            d.draw_circle(400, 300, 50, ColorRydit::Rojo);
        }
        
        // Rust decide cuándo parar
        if escape {
            break;
        }
    }
}
```
