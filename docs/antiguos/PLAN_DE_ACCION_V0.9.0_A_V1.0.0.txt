## 🌳 ÁRBOL DE TAREAS - PRIORIDADES REVISADAS

### FASE 0: GPU INSTANCING (PRIORIDAD MÁXIMA) ⭐⭐⭐⭐⭐

**¿Por qué primero?**
- Sin GPU Instancing → NO hay escenas bíblicas
- Sin GPU Instancing → NO hay visualización científica masiva
- Render Queue es INSUFICIENTE (1000 vs 10,000+ necesarias)

**Tareas:**
```
0.1 FFI OpenGL (gl-rs crate)
    - Agregar gl = "0.14.0" a Cargo.toml
    - Crear crates/rydit-gfx/src/gl_instancing.rs
    - Compilar shader GLSL (vertex + fragment)
    - Crear VAO, VBO
    - glDrawArraysInstanced()

0.2 Shaders GLSL Básicos
    - Vertex: pos + color + size (instancing)
    - Fragment: círculo con glow
    - Uniforms: screen_size, time

0.3 Integración con RyDit
    - gpu::emit(x, y, count, color)
    - gpu::render()
    - Fallback a Render Queue si no hay GPU

0.4 Demo: 10,000 Partículas
    - Explosión de partículas
    - 60 FPS estables
    - Comparativa: antes (1000) vs después (10,000)
```

**Tiempo**: 4-6 semanas
**Riesgo**: Alto (FFI + unsafe)
**Recompensa**: 10x más partículas

### FASE 2: N-BODY GRAVITY ⭐⭐⭐⭐

**Para escenas de creación/sistema solar.**

**Tareas:**
```
2.1 N-body gravity básico
    - crates/rydit-physics/src/nbody.rs
    - F = G * m1 * m2 / r^2
```

**Decisión**: **AHORA. Sin excusas.**

### 2. ¿Versión 0.0.1 o 1.0.0?

**0.0.1 (RECOMENDADO)**:
- ✅ Honesto sobre el estado
- ✅ Sin presión de "versión 1"
- ✅ GPU Instancing como "nuevo inicio"

**1.0.0**:
- ❌ Falsa expectativa de completitud
- ❌ Presión innecesaria
