## 📖 Contexto Histórico

### Sesión 16-17 (v0.1.1-v0.1.2): La Propuesta del Binding Universal

**Situación:**
- 200+ líneas de ifs anidados en `evaluar_expr()`
- Cada builtin nuevo requería modificar el core
- Sin tests unitarios para funciones individuales
- Código difícil de mantener

**Propuesta (BINDING_UNIVERSAL_DISENO.txt):**
```
Crear crate `rydit-bindings` con registry de funciones
- BindingFn = fn(&[Valor]) -> Result<Valor, String>
- Registry dinámico y extensible
- Plugins externos posibles
- 180 líneas menos en el core
```

**Decisión en v0.1.3:**
> ❌ **POSTERGADO** - "Demasiado temprano, primero madurar features básicas"

---

## 🎨 Contexto Histórico: migui (raygui)

**Situación:**
- v-shield solo tiene primitivas básicas (círculos, rects)
- No hay GUI para herramientas de desarrollo
- raylib tiene raygui (immediate mode GUI)
- Usuario necesita herramientas visuales

**Propuesta (QWEN.md):**
```
migui API:
- gui.button(), gui.label(), gui.checkbox()
- gui.slider(), gui.textbox(), gui.panel()
- Immediate mode: retorna bool/valor por frame
- ~1000 líneas, 1-2 semanas
- v0.4.0
```

**Decisión:**
> ✅ **APROBADO para v0.4.0** - "Inmediato mode GUI es la arquitectura correcta"

### 2. migui con raygui (Immediate Mode) ✅

**Razones para immediate mode GUI:**

| Característica | Immediate Mode (raygui) | Retained Mode (Qt-style) | ¿Por qué immediate? |
|----------------|------------------------|--------------------------|---------------------|
| **Estado** | Sin estado complejo | Estado interno complejo | ✅ Simple para RyDit |
| **Código** | ~1000 líneas | ~5000+ líneas | ✅ Cabe en v0.4.0 |
| **Dependencias** | raylib (ya está) | Qt/ImGui (pesado) | ✅ Sin deps nuevas |
| **Curva aprendizaje** | 1 día | 1 semana | ✅ Usuario Android |
| **Integración** | Nativa con raylib | Wrapper complejo | ✅ v-shield existe |

**Arquitectura migui (v0.4.0):**
```rust
// Immediate mode GUI - cada frame se dibuja todo
pub struct Migui {
    // Sin estado complejo, solo configuración
    pub theme: Theme,
    pub font: Font,
}

impl Migui {
    // Cada llamada retorna el estado ACTUAL
    pub fn button(&mut self, text: &str, rect: Rect) -> bool {
        // 1. Dibujar botón
        // 2. Verificar click
        // 3. Retornar estado (presionado/no presionado)
        return is_clicked;
    }

    pub fn label(&mut self, text: &str, rect: Rect) {
        // Solo dibuja, sin estado
    }

    pub fn checkbox(&mut self, text: &str, checked: bool) -> bool {
        // Retorna nuevo estado de checked
        return !checked;
    }
}
```

**Ejemplo de uso en RyDit:**
```rydit
import migui

shield.init

# Cada frame se ejecuta esto
rytmo game_loop {
    # Ventana principal
    migui::window("Mi App", 800, 600, {
        # Panel lateral
        migui::panel(10, 10, 200, 580, {
            migui::label("Puntuación: " + score)
            migui::label("Vidas: " + vidas)
            
            onif migui::button("Restart", 50, 500, 100, 40) {
                restart_game()
            }
        })
        
        # Área principal
        migui::panel(220, 10, 570, 580, {
            # Contenido dinámico del juego
            draw_game()
        })
    })
}
```

**¿Por qué es la decisión CORRECTA?**

1. ✅ **Filosofía RyDit** - Simple, directo, sin estado complejo
2. ✅ **Arquitectura existente** - v-shield ya usa raylib
3. ✅ **Tamaño razonable** - ~1000 líneas vs ~5000 de ImGui
4. ✅ **Usuario objetivo** - Android/Termux, sin X11 complejo
5. ✅ **Immediate mode** - Coincide con game loop de Snake

## 💡 Lecciones Aprendidas

### 1. No Refactorizar Prematuramente

**Binding Universal:**
- ❌ v0.1.1: "200 líneas de ifs, hay que refactorizar YA"
- ✅ v0.1.6: "35 builtins, manejable, esperar a 50+"

**Lección:**
> Refactorizar cuando el dolor es REAL, no anticipado

**Señales de que es hora:**
- El código es INMANEJABLE (no solo "feo")
- Los tests son IMPOSIBLES de escribir
- Los usuarios piden features que no puedes agregar

### 2. Arquitectura Correcta > Arquitectura Completa

**migui con raygui:**
- ❌ ImGui completo: 5000+ líneas, dependencias C++
- ✅ raygui style: 1000 líneas, solo raylib

**Lección:**
> 80% de features con 20% de complejidad = Éxito

**Señales de arquitectura correcta:**
- Coincide con filosofía del proyecto
- Usa dependencias existentes
- Tamaño razonable para el timeline
- Usuarios pueden entenderlo rápido

### 3. El Contexto Importa

**Android/Termux cambia todo:**
- ❌ Qt/ImGui - Requieren X11, compilación compleja
- ✅ raygui - Solo raylib (ya compilado)
- ❌ Binding Universal - Complejidad innecesaria
- ✅ Features concretas - strings, io, random, time

**Lección:**
> La plataforma objetivo DEFINE la arquitectura
